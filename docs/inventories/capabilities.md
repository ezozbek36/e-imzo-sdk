# Capability Inventory

## How to use this file

- This is a descriptive capability map across the currently observed local, server, mobile, and editorial layers.
- Capability boundaries here are intentionally conservative; some areas are version-sensitive, deployment-sensitive, or still ambiguous.

|---------------------------------------------|--------------------------------|---------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------|--------------------|--------------------------------------------------------------------------------------------------------------------------|

## Cross-cutting notes

- Local helper availability drifts by client version; older local verify/timestamp helpers should not be treated as a stable baseline.
- The local-vs-server responsibility split for verification, timestamping, and certificate-status work is still incomplete.
- Mobile flow shape is observed as `init -> poll -> finalize`, but backend finalize method details remain partly conflicted.
- Field-name drift around `challenge` / `challange` and `pkcs7b64` / `pkcs7_64` should stay visible while capability mapping evolves.
