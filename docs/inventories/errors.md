# Error Inventory

## How to use this file

- This is an editorial inventory of failure surfaces collected across the current RFC set and source corpus.
- It complements RFC-0006, but it is not a final SDK error taxonomy or a complete status-code reference.

## Error inventory notes

- Do not over-normalize local textual errors into a numeric schema that the current sources do not provide.
- Do not treat mobile status codes as a complete final taxonomy; several remain deployment- or log-dependent.
- Do not merge operational dependency failures into protocol guarantees just because they recur in practice.
- Do not finalize retriable-vs-terminal behavior before more fixture-driven validation exists.
